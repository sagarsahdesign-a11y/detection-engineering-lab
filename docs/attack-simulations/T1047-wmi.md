# T1047 — Windows Management Instrumentation

| Field | Value |
|-------|-------|
| **ATT&CK ID** | T1047 |
| **Tactic** | Execution |
| **Simulation Date** | 2025-06-10 |
| **Status** | ✅ Detected |

---

## ATT&CK Description

Adversaries may abuse Windows Management Instrumentation (WMI) to execute malicious commands and payloads. WMI is designed for programmers and is the infrastructure for management data and operations on Windows-based operating systems.

---

## Attack Objective

Execute arbitrary commands via WMI, simulating fileless execution and remote process creation commonly used in lateral movement and persistence.

---

## Atomic Red Team Test

| Field | Value |
|-------|-------|
| **Test Number** | 1 |
| **Command** | `Invoke-AtomicTest T1047 -TestNumbers 1` |

### Test Commands

```cmd
wmic process call create "cmd.exe /c calc.exe"
```

Or PowerShell:

```powershell
Invoke-WmiMethod -Class Win32_Process -Name Create -ArgumentList "notepad.exe"
```

### Prerequisites

- [x] Process creation logging enabled
- [x] WMI service running (default on Windows)

---

## Expected Windows Logs

| Event Source | Event ID | Description |
|--------------|----------|-------------|
| Sysmon | 1 | Process Creation (WmiPrvSE.exe children) |
| Microsoft-Windows-WMI-Activity | 5857-5861 | WMI operational events |

### Key Indicators

- `wmic process call create`
- Parent: `WmiPrvSE.exe`
- `Invoke-WmiMethod`, `Get-WmiObject` in PowerShell 4104

### Sample Log Excerpt

```

ParentImage: C:\Windows\System32\wbem\WmiPrvSE.exe
```

## Detection Gaps

- SCCM and monitoring tools use WMI legitimately
- Remote WMI from other hosts may not show wmic.exe on victim (remote execution)
- WMI event subscriptions (persistence) are separate from execution detection

---

## Remediation Recommendations

1. Restrict WMI permissions via GPO for non-admin users
2. Baseline WMI consumers in environment
3. Monitor for remote WMI connections (port 135/445)
4. Add detection for WMI Event Subscription persistence (T1546.003)
